vue3.0+TypScript (篇章一)

先做准备工作吧

大致了解vue2.0和vue3.0的对比

  • vue 2.0 采用的是flow进行编写，而3.0源码全部采用typescript开发的 对typescript 有更好的支持
  • 源码题体积优化移除了部分api，例如filter。使用函数式api更好的tree-shaking
  • CompositionAPI：整个业务代码的逻辑，提取公共逻辑（vue2采用的mixin-容易有命名冲突）
  • 数据劫持的优化：vue3.0采用的是proxy 代理的方式管理数据，不用再通过defineProperty层层递归增加拦截器，（所以vue2我们尽量让数据扁平化）。
  • 编译优化vue3实现了静态模板分析，重写了diff算法，静态节点不参与比对
  • 自定义渲染器：可以用来创建自定义的渲染器，改写vue底层渲染逻辑，可以自定义定制例如 webGl、小程序、canvas等渲染元素
  • 新增Fragment、Teleport、Suspense组件

简单了解ts常用的几个基本用法

  • 基本数据类型
  • 接口interface
  • object 类型
  • 函数类型
