crodova APP篇目一

项目环境搭建

  1. 安装node.js环境
  2. 用npm下载模块工具包:
  • 在OS X和Linux上：

-g标志是告诉 npm 我们全局安装 cordova。否则我们将会安装在当前工作目录的 node_modules子目录。

安装完成后，你应该能够在命令行中运行cordova命令，在没有任何参数的时候会打印一些帮助信息。

创建app

  1. 命令行cd到你维护源代码的目录中，并创建你的cordova项目：$ cordova create hello com.example.hello HelloWorld
  2. 这将会为你的cordova应用创造必须的目录。默认情况下，cordova create命令生成基于web的应用程序的骨骼，项目的主页是 www/index.html 文件。

    添加一个平台

  3. 在创建完一个Cordova项目后，导航到项目目录。在项目目录中，你需要添加你想构建app的平台

添加平台

为了添加平台， 键入 cordova platform add <platform name>。

为了知道你可以添加的平台，你可以运行cordova platform。

运行App

在命令中,键入cordova run <platform name>。

hexo安装步骤

Posted on 2017-10-24 | In blog

安装前序

安装之前确定自己电脑是否下面两个应用程序：

  1. Node.js
  2. Git
    可以在终端输入node -v检查Node.js是否安装成功。
    输入git -v检查Git是否安装成功。

    安装hexo

  3. 电脑里面新建一个文件夹。我的文件目录是 /Users/emily/Documents/hhxblog
    在Terminal或者window终端cd到该路径。之后的hexo 指令都是在此路径下输入的。

输入：$ cd /Users/emily/Documents/hhxblog

  1. 使用npm完成安装。

    $ npm install -g hexo-cli
    $ npm install hexo --save

  2. 然后在命令窗口输入hexo指令:hexo -v查看是否安装成功。
  3. 本地博客用起来。
    初始化安装hexo
    $ hexo init
    $ npm install
    构建hexo博客框架文件
    $ hexo g
    在本地浏览器运行
    $ hexo s
    在浏览器中打开:http://localhost:4000/ ,就会出现

关联GitHub建立站点

前提，你的申请一个Github账号
1.注册Git账号
2.新建仓库 New repository

3.填写信息，注意图中重点。

4.开启gh-pages功能

点击界面右侧的Settings，你将会打开这个库的setting页面，向下拖动，直到看见GitHub Pages，如图：

点击Automatic page generator，Github将会自动替你创建出一个gh-pages的页面。 如果你的配置没有问题，那么大约15分钟之后，yourname.github.io这个网址就可以正常访问了~ 如果yourname.github.io已经可以正常访问了，那么Github一侧的配置已经全部结束了。

配置Deployment

1.首先，你需要为自己配置身份信息，打开命令行，然后输入：

$ git config --global user.name "yourname"
$ git config --global user.email "youremail"

3.如果使用git方式进行部署，执行npm install hexo-deployer-git –save来安装所需的插件
然后在当前目录打开命令行，输入：
$ hexo d
随后按照提示，分别输入自己的Github账号用户名和密码，开始上传。 然后通过http://yourname.github.io/ 来访问自己刚刚上传的网站。

前端开发学习路线图

发表于 2017-10-23 | 分类于 学习

[TOC]

一、知彼

  • 要有一个清晰学习路线图，先得对前端有大致的了解所以先借鉴了一下国外一大佬分享的学习路线图。

二、知己

  • 了解了前端的大致流派，那么接下来就是对自己前端方面的了解，初步评估自己还处于gettingDeeper的徘徊路口。

  • 先说一下自己以后想要形成的项目开发风格：webpack+vue+sass

  • 我总结了现目前自己需要恶补几个问题：
    1.代码规范
    2.css模块化开发，js组件化开发
    3.web响应式开发如何做好自适应。
    4.浏览器低版本兼容问题。
    5.如何做好手机端webApp适配。
    6.项目部署。
    7.等等。。。

    三、战略

  • 结合这么些年的失败经验来说，学习东西没有那种方法是比实践来得更快的。编程就是做项目！但是做也要分怎么做才算是高效的做。

  1. 时间管理：每天任务用时间四象大致限划分，再着手去做。参考GTD时间管理理念。
  2. 对于基础薄弱的我来说，在遇到问题的时候尽量多问，就是驱动式学习，先自己思考–>google–>问同行技术大牛的,再不行读源码！再不行，先放着，做点儿其他的，说不定就有思路了。
  3. 找环境。一定要在一个奋发向上的环境下学习工作，如果有能力和nb的同事共事那是最好不过的了。
  4. 多敲代码！！！

四、短期计划

  • 1.今天笔试的没答上来的题去了解了解，写个笔记。
  • 2.在GitHub上fork一个vue.js+webpack的项目，调试研读别人的代码，借鉴人家好的编程习惯和逻辑。
  • 3.用vue.js+webpack把现在接的项目完成。
  • 4.了解如何完成一个vue.js+webpack的完整项目开发，比较困惑的是写好的项目如何部署到服务器，直到输入网址可以直接访问。
  • 5.有时间多多看一些逻辑比较复杂的js实现代码。
  • 6.在项目中将自己前说的遗留的问题解决掉。

五、来日方长

等到上面的计划都完成了（当然其中也会随之出现很多没有考虑到的问题），我感觉我基本也能真正意义上的独立开发一个前端项目了。
我以后会做的就会有：
1.vue.js+webpack+node.js+mongoDB
2.vue.js源码阅读。
3.英语的提升（平日多用Google看英文文献）
4.优化代码，提炼代码，使项目开发框架化开发。
5.管理能力。

总结

‘去做’ 才是硬道理。希望一年后看到一个nb的自己。加油~
