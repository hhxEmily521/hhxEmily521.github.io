前端开发学习路线图

发表于 2017-10-23 | 分类于 学习

[TOC]

一、知彼

  • 要有一个清晰学习路线图，先得对前端有大致的了解所以先借鉴了一下国外一大佬分享的学习路线图。

二、知己

  • 了解了前端的大致流派，那么接下来就是对自己前端方面的了解，初步评估自己还处于gettingDeeper的徘徊路口。

  • 先说一下自己以后想要形成的项目开发风格：webpack+vue+sass

  • 我总结了现目前自己需要恶补几个问题：
    1.代码规范
    2.css模块化开发，js组件化开发
    3.web响应式开发如何做好自适应。
    4.浏览器低版本兼容问题。
    5.如何做好手机端webApp适配。
    6.项目部署。
    7.等等。。。

    三、战略

  • 结合这么些年的失败经验来说，学习东西没有那种方法是比实践来得更快的。编程就是做项目！但是做也要分怎么做才算是高效的做。

  1. 时间管理：每天任务用时间四象大致限划分，再着手去做。参考GTD时间管理理念。
  2. 对于基础薄弱的我来说，在遇到问题的时候尽量多问，就是驱动式学习，先自己思考–>google–>问同行技术大牛的,再不行读源码！再不行，先放着，做点儿其他的，说不定就有思路了。
  3. 找环境。一定要在一个奋发向上的环境下学习工作，如果有能力和nb的同事共事那是最好不过的了。
  4. 多敲代码！！！

四、短期计划

  • 1.今天笔试的没答上来的题去了解了解，写个笔记。
  • 2.在GitHub上fork一个vue.js+webpack的项目，调试研读别人的代码，借鉴人家好的编程习惯和逻辑。
  • 3.用vue.js+webpack把现在接的项目完成。
  • 4.了解如何完成一个vue.js+webpack的完整项目开发，比较困惑的是写好的项目如何部署到服务器，直到输入网址可以直接访问。
  • 5.有时间多多看一些逻辑比较复杂的js实现代码。
  • 6.在项目中将自己前说的遗留的问题解决掉。

五、来日方长

等到上面的计划都完成了（当然其中也会随之出现很多没有考虑到的问题），我感觉我基本也能真正意义上的独立开发一个前端项目了。
我以后会做的就会有：
1.vue.js+webpack+node.js+mongoDB
2.vue.js源码阅读。
3.英语的提升（平日多用Google看英文文献）
4.优化代码，提炼代码，使项目开发框架化开发。
5.管理能力。

总结

‘去做’ 才是硬道理。希望一年后看到一个nb的自己。加油~
